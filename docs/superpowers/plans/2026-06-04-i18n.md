# i18n (pl/en/de) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the whole app (React UI, Rust tray menu, user-facing backend errors) available in Polish, English and German, with OS auto-detect on first run and a Settings selector.

**Architecture:** `react-i18next` with three JSON resource files as the single source of truth. The frontend pushes translated tray labels to a Rust command that caches them and rebuilds the native tray; user-facing backend errors return stable keys the frontend maps to `t("errors.*")` (raw text fallback). `ui_language` persists in `config.json` (merge-safe `save_config`).

**Tech Stack:** React 19 + TypeScript (strict), Tailwind v4, react-i18next + i18next, Tauri 2 (Rust tray).

## Translation conventions (apply in every extraction task)

- One i18next namespace (`translation`); keys nested by area. Key style: `area.camelCaseName` (e.g. `settings.interfaceLanguage`, `usage.totalTranscriptions`).
- Replace a JSX string `Foo` with `{t("area.foo")}`; a prop string `placeholder="Foo"` with `placeholder={t("area.foo")}`. Add `const { t } = useTranslation();` at the top of the component if not present.
- Dynamic values use interpolation: `t("models.downloading", { index, total, file, pct })` with the en value `"Downloading {{index}}/{{total}}: {{file}} ({{pct}}%)"`.
- Counts use i18next plurals: define `key_one` / `key_other` (en/de) and `key_one`/`key_few`/`key_many`/`key_other` (pl), called as `t("key", { count })`.
- Every key added to `en.json` MUST be added to `pl.json` and `de.json` in the same task (EN = original wording; PL/DE = natural, technically-correct translations, no emoji).

**Files:** Create `src/i18n/detect.ts`

- [ ] **Step 1: Write the module**

```ts
export const SUPPORTED_LANGUAGES = ["en", "pl", "de"] as const;
export type Language = (typeof SUPPORTED_LANGUAGES)[number];

export function isSupported(lang: string): lang is Language {
  return (SUPPORTED_LANGUAGES as readonly string[]).includes(lang);
}

// First-run default: derive from the OS/webview locale, else English.
export function detectOsLanguage(): Language {
  const raw = (navigator.language || "en").slice(0, 2).toLowerCase();
  return isSupported(raw) ? raw : "en";
}
```

### Task 5: Persisted-language sync (apply + change + persist)

**Files:** Create `src/i18n/language.ts`

- [ ] **Step 1: Write the module**

```ts
import { invoke } from "@tauri-apps/api/core";
import i18n from "./index";
import { isSupported, Language } from "./detect";
import { pushTrayLabels } from "./tray";

// Read the persisted ui_language from config.json. If present, apply it. If absent
// (first run), persist whatever the OS-detected default resolved to at init.
export async function applyPersistedLanguage(): Promise<void> {
  try {
    const cfg = JSON.parse((await invoke<string>("load_config")) || "{}");
    const saved = cfg.ui_language;
    if (typeof saved === "string" && isSupported(saved)) {
      if (i18n.language !== saved) await i18n.changeLanguage(saved);
    } else {
      await persistLanguage(i18n.language as Language);
    }
  } catch (err) {
    console.error("i18n: failed to apply persisted language:", err);
  }
  await pushTrayLabels();
}

async function persistLanguage(lang: Language): Promise<void> {
  const cfg = JSON.parse((await invoke<string>("load_config")) || "{}");
  cfg.ui_language = lang;
  await invoke("save_config", { config: JSON.stringify(cfg) });
}

export async function changeLanguage(lang: Language): Promise<void> {
  await i18n.changeLanguage(lang);
  await persistLanguage(lang);
  await pushTrayLabels();
}
```

If implementing strictly in order, do Task 9 before linting Task 5, or temporarily stub `pushTrayLabels`. The recommended order is 1→2→3→4→7→8→9→5→6 so `tray.ts` exists first.)

### Task 9: Frontend tray-label push

**Files:** Create `src/i18n/tray.ts`

- [ ] **Step 1: Write the module**

```ts
import { invoke } from "@tauri-apps/api/core";
import i18n from "./index";

// Build the tray label set from the current language and hand it to the backend,
// which caches it and rebuilds the native tray. Tray translations therefore live
// in the same JSON files as the rest of the UI.
export function pushTrayLabels(): Promise<void> {
  const t = i18n.t.bind(i18n);
  const labels = {
    start_recording: t("tray.startRecording"),
    stop_recording: t("tray.stopRecording"),
    copy_last: t("tray.copyLast"),
    usage: t("nav.usage"),
    models: t("nav.models"),
    history: t("nav.transcriptions"),
    settings: t("nav.settings"),
    lock_window: t("tray.lockWindow"),
    unlock_window: t("tray.unlockWindow"),
    select_microphone: t("tray.selectMicrophone"),
    default_microphone: t("tray.defaultMicrophone"),
    quit: t("tray.quit"),
  };
  return invoke("set_tray_labels", { labels }).catch((err) =>
    console.error("i18n: failed to push tray labels:", err),
  );
}
```

### Task 18: Key-parity check

**Files:** Create `scripts/check-i18n-keys.mjs`

- [ ] **Step 1: Write the script**

```js
import en from "../src/i18n/locales/en.json" with { type: "json" };
import pl from "../src/i18n/locales/pl.json" with { type: "json" };
import de from "../src/i18n/locales/de.json" with { type: "json" };

function flatten(obj, prefix = "") {
  return Object.entries(obj).flatMap(([k, v]) =>
    v && typeof v === "object" && !Array.isArray(v)
      ? flatten(v, `${prefix}${k}.`)
      : [`${prefix}${k}`],
  );
}
const sets = { en: new Set(flatten(en)), pl: new Set(flatten(pl)), de: new Set(flatten(de)) };
let ok = true;
for (const base of ["en"]) {
  for (const other of ["pl", "de"]) {
    const missing = [...sets[base]].filter((k) => !sets[other].has(k));
    const extra = [...sets[other]].filter((k) => !sets[base].has(k));
    if (missing.length) { ok = false; console.error(`Missing in ${other}:`, missing); }
    if (extra.length) { ok = false; console.error(`Extra in ${other} (not in en):`, extra); }
  }
}
if (!ok) { console.error("i18n key parity FAILED"); process.exit(1); }
console.log("i18n key parity OK");
```

- [ ] **Step 2: Add an npm script** to `package.json`: `"check:i18n": "node scripts/check-i18n-keys.mjs"`.
