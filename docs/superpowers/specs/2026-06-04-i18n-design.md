Date: 2026-06-04
Status: Approved for planning
Scope: Full app translation for Polish (pl), English (en), German (de) â€” React UI, the Rust tray menu, and user-facing backend errors.

## 1. Goal

Make the whole Simplevoice app available in Polish, English, and German: every
user-facing string in the React UI, the native tray menu (built in Rust), and the
backend errors that surface in the UI. English is the fallback. The brand name
"Simplevoice" is never translated.

## 2. Decisions (from brainstorming)

- **Library:** `react-i18next` with JSON resource files â€” the single source of truth
  for all translations (UI and tray).
- **Scope:** whole app â€” React views/onboarding/dialogs + the Rust tray menu + the
  user-facing backend errors.
- **Default language:** on first run, detect the OS locale (`navigator.language`),
  map to pl/de/en, fall back to en, and persist. A separate "Interface Language"
  selector in Settings (distinct from the existing "Transcription Language").
- **Persistence:** `ui_language` in `config.json` (relies on the merge-safe
  `save_config`).

## 3. Architecture & file structure

```
src/i18n/
  index.ts        # react-i18next init: resources, fallbackLng "en",
                  # supportedLngs ["en","pl","de"], interpolation escapeValue:false
  locales/en.json # single source of truth â€” all EN strings
  locales/pl.json
  locales/de.json
  tray.ts         # builds the tray label object from t() and invokes set_tray_labels
  detect.ts       # OS language detection (navigator.language -> pl/de/en, fallback en)
```

- `main.tsx` imports `./i18n` before render (i18next is a global singleton); the app
  may be wrapped in `<I18nextProvider>` but a plain import is sufficient for CSR.
- Components use `const { t } = useTranslation()` and `t("settings.title")`. No inline
  user-facing strings remain.
- One namespace, keys nested by area: `common`, `nav`, `titlebar`, `usage`, `models`,
  `transcriptions`, `settings`, `onboarding`, `tray`, `errors`, `updater`,
  `recordingWindow`, `hud`, `languages`.

## 4. Language selection & persistence flow

- **Startup:** read `ui_language` from `config.json`. If set, use it. If absent (first
  run), detect via `navigator.language` mapped to pl/de/en (fallback en), set it via
  `i18n.changeLanguage`, and persist it to `config.json`.
- **Settings:** a new, dedicated "Interface Language" section (its own card, placed
  first in the settings grid) with a select showing endonyms: Polski / English /
  Deutsch. Separate from "Transcription Language" (`asr_language`).
- **Change:** `i18n.changeLanguage(lang)` + `updateConfig("ui_language", lang)` + push
  the tray labels. react-i18next re-renders the whole UI immediately.

## 5. Translation key organization & coverage

- Full coverage: all views (Usage, Models, Transcriptions, Settings, RecordingWindow),
  onboarding (8 step titles/bodies + step counter + buttons), Sidebar, TitleBar,
  Updater, the recording/transcribing HUD, dialogs, toasts.
- **Interpolation** for dynamic content: `"Step {{current}} / {{total}}"`,
  `"Downloading {{index}}/{{total}}: {{file}} ({{pct}}%)"`, app version, device counts.
- **Pluralization** via i18next plural keys where counts appear (e.g. Usage counters,
  "X transcriptions").
- Numbers/dates formatted via `Intl` with the active locale (Usage stats).
- "Simplevoice" is always a literal, never a translation key.
- The selector lists endonyms; `languages` keys hold display names.

## 6. Tray (Rust) localization

- New Rust state `TrayLabels` (a struct of ~12 fields: both toggle variants
  `start_recording`/`stop_recording`, `copy_last`, `usage`, `models`, `history`,
  `settings`, `quit`, `default_mic`, `lock`/`unlock`), held in a `Mutex` in app state
  with English defaults.
- New command `set_tray_labels(labels)` â€” the frontend calls it on startup and on
  language change; it stores the labels and calls `rebuild_tray_menu`.
- `rebuild_tray_menu_inner` reads labels from state instead of hardcoded literals; the
  toggle item picks the start/stop variant from the recording state.
- **Cold start:** Rust builds the tray with English defaults; the frontend (which runs
  even while the main window is hidden) pushes labels within ~300 ms, triggering a
  localized rebuild. The menu is only seen on click, so there is no visible flash.
  Tray translations live in the same JSON â€” no duplication in Rust.

## 7. Backend error localization

- Raw `Err("...")` strings from Rust are English. Rather than translate every internal
  error, cover the ones that actually reach the UI: the "Configuration Required" dialog
  (recording-failed-to-start), model load failure, download/convert failures, shortcut
  registration errors.
- Mechanism: these user-facing errors return a **stable key** (e.g.
  `error.no_model_loaded`, `error.model_load_failed`); the frontend maps them via
  `t("errors.<key>")`. Unmapped/technical errors **fall back to the raw text** (English,
  but nothing breaks).
- This translates everything the user actually sees without rewriting the backend.

## 8. OS locale detection

- `navigator.language` in the webview (reflects the OS locale under Tauri); take the
  prefix and map `pl`/`de`/`en`, else fall back to `en`. No extra dependency
  (`@tauri-apps/plugin-os` rejected to avoid a needless dep).

## 9. Testing & verification

- `pnpm lint` (tsc; react-i18next ships types) + `pnpm build` + `cargo check` (tray
  changes).
- **Key parity:** a small script/step asserts that `en/pl/de` share an identical key
  set (no missing or extra translations) â€” essential for full coverage.
- Adversarial verification (workflow): no leftover hardcoded user-facing strings in
  components; correct interpolation/plural usage; PL/DE translation quality
  (naturalness, technical terminology, no emoji, "Simplevoice" untouched).
- Manual: changing the language in Settings updates every view and the tray.

## 10. Out of scope

- RTL languages (pl/de/en are all LTR).
- Translating user content (the transcriptions themselves).
- The brand name "Simplevoice".
- Fully translating every deep internal Rust error (only the user-facing ones).
